
Problem/ Curiosity:  
For my project I am interested in exploring the parallel between computer memory management and human memory storage. Just as C and  C++ require careful consideration of how and where data is stored in memory through concepts like allocation, deallocation, and memory mapping, human memories also exist in complex, interconnected patterns that shift and fade over time. The projects address the question: How can we visualize and interact with memory - both digital and human - in ways that reflect its dynamic, impermanent nature?


First Step:
The first approach uses a gui interface with a fluid, non-grid layout to create a collective memory map, allowing users to contribute and visualize memories tied to specific locations or prompts. Users are asked prompts and input text or image or sound that represents memory.

The second approach uses LED matrices to physically represent memory through falling letters and changing colors, making abstract concepts of memory storage tangible and visible. Both projects use C++ to bridge the technical implementation of memory management with artistic metaphors about how we store, access, and lose our memories over time.
