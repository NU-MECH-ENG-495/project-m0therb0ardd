63// Internal representation of the frame-buffer that as well can
64// write itself to GPIO.
65// Our internal memory layout mimicks as much as possible what needs to be

67class Framebuffer {
68public:
69 // Maximum usable bitplanes.
70 //
71 // 11 bits seems to be a sweet spot in which we still get somewhat useful
72 // refresh rate and have good color richness. This is the default setting
73 // However, in low-light situations, we want to be able to scale down
74 // brightness more, having more bits at the bottom.
75 // TODO(hzeller): make the default 15 bit or so, but slide the use of
76 // timing to lower bits if fewer bits requested to not affect the overall
77 // refresh in that case.
78 // This needs to be balanced to not create too agressive timing however.
79 // To be explored in a separete commit.
80 //
81 // For now, if someone needs very low level of light, change this to
82 // say 13 and recompile. Run with --led-pwm-bits=13. Also, consider
83 // --led-pwm-dither-bits=2 to have the refresh rate not suffer too much.
84 static constexpr int kBitPlanes = 11;
85 static constexpr int kDefaultBitPlanes = 11;
